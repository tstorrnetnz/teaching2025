Hello World

Some final thoughts about this project and where it could be taken

  1. The database also has a one-to-many relationship (sauce) - this might be enough for Y12s
  2. There are descriptions for toppings - this could be extended (GF, DF, photos etc)
  3. The entire Flask app code is in one file (main.py) - config (at the very least) belongs in its own file - See Teddy Bears Picnic for some ideas around that
  4. There are no photos in this version - again, see TBP for ideas around that - photos can be stored as BLOBs in the database - but probably shouldn't be in this instance - use the STATIC folder
  5. The static folder is the only part of any Flask app that users (ie: the browser) can access, everything else is secured and only available to the Flask app - so css, js, images, logos etc all need to be in there.
  6. I have used inline CSS - this is generally a very bad idea - separation of concerns... content should be in HTML, style in CSS. This is not a web design course, its all about the Flask.
  7. Finally - why Pizzas? - I use this as a (and I hate this term, but I'm tired and couldn't think of another one) 'Team Building Exercise' (YAY!) - everyone chips in $5 and we order a tonne of pizza. Not everyone can afford it - I normally find myself $30 or so out of pocket #teacherlife but its a good chance to get photos of pizzas to explore the use of images in a Flask app.
